Why SSH was the right pivot

HTTPS push failed because password auth is no longer supported. We created an SSH key, added it to GitHub, trusted the GitHub host key locally, and moved the repository remote to the SSH URL.

Initial remote
`https://github.com/Doolannn/Material-Design.git`
Working remote

Why this matters for Figma later

A GitHub-backed component inventory makes the repository the single implementation source. That means your future Figma connection can point to real components and stories instead of a duplicated design-only catalog.

01

Verified the local repository

The workspace already had `.git`, but no remote was configured yet.

 02

Attached `origin`

This created the direct path from the local workspace to the GitHub repository.

 03

Switched to SSH

HTTPS failed, so SSH became the durable authentication route.

 04

Created and registered a key

The public key was added to GitHub so pushes could authenticate cleanly.

 05

Trusted GitHub’s host key

This resolved SSH host verification on the machine.

 06

Fetched and merged remote history

The repo already had existing commits, so histories had to be reconciled.

 07

Pushed back to `main`

From there, repo-backed pages and component previews could be published and reviewed.
